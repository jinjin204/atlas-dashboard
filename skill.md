# User Skill Profile
- **Role**: Product Owner / Manager
- **Capabilities**: ビジネスロジックの決定、Google Drive上のファイル管理。
- **Non-Capabilities**: Pythonコードの直接修正、複雑なエラーログのデバッグ。
- **Support Strategy**: AIは技術的な詳細をブラックボックス化し、ユーザーには「ビジネス判断が必要な選択肢」のみを提示せよ。

## AI開発者 規律ルール

### UI統合時のJSデバッグ規律
- **Python側の修正だけで完了としない。** UI統合時は、JS側のデバッグログをシミュレートし、コンソールエラーの有無を予測して報告せよ。
- 具体的チェック項目:
  1. `json.dumps()` の出力が、JS側の期待する形式（フィールド名・型・ネスト構造）と完全一致するか突き合わせる。
  2. `ensure_ascii=False` で出力した日本語文字列が、JSのテンプレートリテラルや文字列連在で構文エラーを起こさないか検証。
  3. `index.html` のインジェクション箇所（`/* INJECTED LOGIC WILL GO HERE */` 等）が正しく残っているか確認。
  4. `window.onerror` が正しく機能するか、関数シグネチャを毎回検証。
- **真っ白画面は敗北。** 原因不明のブランク画面を許容せず、必ずエラーメッセージを表示する仕組みを維持せよ。

### パイプライン完全性の規律
- **「表示できた」で完了としない。** 「データが次のステップに流れるか」まで検証せよ。
- 具体的チェック:
  1. 関数Aが関数Bを呼んでいる → 関数Bは実在するか？ (drive_utils断絶の教訓)
  2. UIのボタンがバックエンド関数を呼んでいる → その関数はapp.pyのどこから呼ばれるか？
  3. Write-backが発生する → 上書き前の値を保存しているか？ ロールバック可能か？
