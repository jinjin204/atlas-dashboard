# Autonomous Debugging Workflow
1. **Detect**: エラー（Traceback）が発生したら、まず原因を「環境・データ・ロジック」に分類せよ。
2. **Scan**: `log_df` や `master_df` の中身を `head()` 等で覗き、データの現状を物理的に確認せよ。
3. **Fix**: 修正コードを作成し、現在の `app.py` への影響をシミュレーションせよ。
4. **Test**: 修正完了の報告前に、内部で `python -c` 等を用いて、重複データを含むサンプルを用いた単体テストを実行し、そのテスト結果（エラーが出ない証明）をチャットに提示すること。
5. **Launch**: アプリを再起動し、成功を確認後、ユーザーにリロードを促せ。
