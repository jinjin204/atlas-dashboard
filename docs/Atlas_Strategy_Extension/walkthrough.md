# アトラス戦略機能拡張（Double Flag System） - 実装完了報告

## 概要
本日、`master_loader.py` および `zeus_chat.py` を更新し、イベントマスタにおける「アクティブ（進軍）」と「表示（監視）」の二重フラグシステムを実装しました。

これにより、**「作業はクリマに集中（アクティブ）しつつ、幕張の締め切りは忘れない（表示）」** という二段構えの管理が可能になりました。

## 変更内容

### 1. Master Loader (`logic/master_loader.py`)
- **アクティブフラグの処理**:
    - `アクティブ`, `Active`, `進軍` カラムを読み取り、`TRUE` の場合のみ生産目標（Target）を合算するようにしました。
- **表示フラグの処理**:
    - `表示`, `Display`, `監視` カラムを読み取り、`TRUE` の場合、そのイベント情報を収集するようにしました。
    - 収集した情報は `data/event_master.json` に保存されます。
- **メタデータの取得**:
    - `応募締切`, `開催日`, `会場`, `ブース`, `搬入` などの情報を取得し、監視リストに含めました。

### 2. Zeus Intelligence (`logic/zeus_chat.py`)
- **監視機能の追加**:
    - 新しく生成される `data/event_master.json` を読み込む機能を追加しました。
- **システムプロンプトの強化**:
    - アクティブな「進軍目標」とは別に、「監視対象イベント」のリストをZeusに認識させました。
    - 特に「応募締切」を意識し、ユーザーに進捗報告と共にリマインドを行うよう指示を追加しました。

##検証結果

以下の検証シナリオを実行し、正常に動作することを確認しました。

### シナリオ: アクティブと表示の分離
ダミーデータを用いて以下の構成でテストしました。

| イベント名 | アクティブ | 表示 | 締切 | 期待される動作 |
|---|---|---|---|---|
| Active Only | TRUE | FALSE | - | 生産目標に合算される。監視リストには出ない。 |
| Display Only | FALSE | TRUE | 2026/06/01 | 生産目標には影響しない。監視リストに出る（Zeusが締切を言及）。 |
| Both Event | TRUE | TRUE | - | 生産目標に合算され、かつ監視リストにも出る。 |

**結果**:
- `production_master.json` の目標数: Active(10) + Both(30) = **40** （Display(20)は除外） -> **OK**
- `event_master.json` の内容: Display Only, Both Event が含まれる -> **OK**
- Zeusプロンプト: "監視対象イベント" セクションに "Display Only" とその締切が含まれていること -> **OK**

## 次のステップ
実際のイベントマスタExcelファイル（`メニュー.xlsx` 内の `イベントマスタ` シート）に、以下のカラムを追加・整備してください。

- **アクティブ**: 計算対象にするイベントに `TRUE` または `1` を入力
- **表示**: Zeusに監視させたいイベントに `TRUE` または `1` を入力
- **応募締切**: 日付を入力（任意）

Zeusはこれを見て、貴方の戦略を補佐します。
